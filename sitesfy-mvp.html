CEO 视角 · 理想产品形态
从「执行自动化」到「判断情报化」
如果重新定义 Sitesfy，核心命题需要一次根本性转变
定位重构
当前：「Proactive Agent 帮你跑出海获客」→ 解决执行问题
理想：「你的第一个出海市场情报官」→ 解决判断问题

出海获客失败的根本原因，90% 不是「没人帮你发邮件」，而是：ICP 选错了、话术不对、时机不对。 执行层的自动化只在判断正确的前提下有价值。判断错误 + 自动化 = 规模化地烧钱。

Layer 1
Signal Intelligence
信号情报层
不是帮你找联系人，是帮你理解市场正在发生什么：谁在找替代方案、竞品哪里出了问题、行业现在的采购信号是什么。
输出：判断依据，不是名单
Layer 2
Judgment Engine
判断引擎
基于信号，给出可执行的具体决策：「这周德国有3家零售商在问 ESG 认证，这是你的入口」「律所有2家高意图访客在搜 GDPR 罚款案例」。
输出：用户的具体行动指令
Layer 3
Execution Autopilot
执行自动驾驶
在判断正确的前提下，执行才值得自动化——发邮件、写文章、跟进序列。这一层大量外包给 Instantly / Apollo。
Sitesfy 只赚「判断正确」的钱

💡 商业模型转变

现在
按功能收费（模块数量）→ 卖工具，客户买的是「能力」
理想
按结果收费：帮你拿到多少条有效回复，每条收费。
这个模型倒逼产品必须真正解决判断问题，而不只是执行问题。
不是「AI 帮你做出海」
是「出海之前，先让你比竞争对手多看一层市场信号」 执行谁都能做，信号差距才是护城河

⚠️ 规模化瓶颈分析：哪些功能无法产品化规模做？

🔴 无法规模化
个性化开发信生成（Content Engine）
每封信需要真正个性化才有效。用模板批量生成后回复率会快速下降 → LLM 生成容易被识别为 AI 味，触发垃圾邮件过滤。瓶颈：邮件质量与发送量之间存在根本性矛盾。发量越大，质量越难保证。
🔴 无法规模化
发件域名预热管理（Outreach Engine）
每个新域名需要 4-8 周预热期，每天限 50-100 封。多客户场景下域名资源是硬上限，无法通过代码解决。瓶颈：物理上受限于 ESP 反垃圾机制，买更多域名成本线性增长。
🔴 无法规模化
访客意图阈值配置（Capture Engine）
「高意图」的定义在不同行业/客户之间差异极大，没有通用标准。每个种子客户需要人工校准 1-2 周才能找到有效阈值。瓶颈：判断逻辑无法泛化，本质是需要人类领域判断。
🟡 规模化困难
SEO 文章内容质量（Content Engine · 律所）
律所内容有专业合规要求，AI 生成内容需人工审核才能发布。规模越大，审核成本越高，无法完全自动化。瓶颈：专业内容 = 人工不可绕过。
🟡 规模化困难
ICP 自动生成准确率（ICP Engine）
URL 扫描 → 自动提炼 ICP 在长尾行业准确率低（如小众制造业、专业服务）。ICP 跑偏了，下游所有模块数据都失真。瓶颈：LLM 对小众垂直行业的理解深度不足，泛化能力有限。
🟡 规模化困难
回复率反哺 ICP 修正（Learning Engine）
飞轮逻辑需要足够的样本量（通常 200+ 封邮件）才有统计意义。小客户发件量不足，反馈信号稀疏，模型修正噪声大于信号。瓶颈：数据驱动优化需要最低数据量门槛，早期客户很难达到。
💡 规模化核心矛盾：Sitesfy 的价值在于「真实个性化」，但规模化依赖「标准化」。
两者天然矛盾——解决路径是找到哪些判断可以被数据替代（规模化），哪些必须保留人工（护城河），而不是试图把所有环节都自动化。

🎲 桌游专属第一期

  • 德语开发信生成
  • 时区智能发送（欧洲时区）
  • Apollo 接入（重点：欧洲桌游零售商条件）

⚖️ 律所专属第一期

  • SEO 文章生成 + 自动发布
  • 关键词研究（出海合规/GDPR方向）
  • RB2B 访客识别接入
  • 高意图访客 → 自动触发外发
  • 内容轻确认流程（律所内容需人审）

★ 两者共用第一期核心

  • URL 扫描 → 自动生成 ICP
  • 个性化英语开发信 + 3步跟进序列
  • 主题行 A/B 变体
  • Instantly API 接入 + 域名预热
  • 发送量限额 + 退订/退信处理
  • 邮件回复率监听 → Proactive 自动调整发量
  • 周报自动生成 + 每周一个决策问题
  • 邮件核心指标采集 + 按维度分析
  • 优化指令输出（Learning → Proactive Core 闭环）

○ 明确延后的

  • 贸易数据（Panjiva）接入
  • LinkedIn 触达
  • 多语言（西/日/法）
  • 跨文化话术精细化
  • 网站嵌入式对话窗口
  • Calendly 预约引导
  • 转化路径追踪（到成单）
  • 月度洞察报告
  • 执行日志界面
  • 回复率反哺 ICP 修正
第一期的本质：用最小可信的一套功能，证明"引擎能自己运转"这件事。
桌游验证 Outbound 路径（找人 → 发邮件 → 自动跟进 → 自动优化方向）；律所验证 Inbound + 捕获路径（SEO 文章 → 吸引访客 → 识别高意图 → 自动触达）。
两个场景加在一起，能覆盖 Proactive Core 的主要信号类型和执行动作。
